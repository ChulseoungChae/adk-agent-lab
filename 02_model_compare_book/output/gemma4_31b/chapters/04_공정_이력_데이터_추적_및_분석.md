# 4. 공정 이력 데이터 추적 및 분석

# 제4장: 공정 이력 데이터 추적 및 분석

## 4.1 공정 이력 관리의 중요성
반도체 공정은 수많은 레시피의 조합으로 이루어집니다. 특정 웨이퍼에서 결함이 발견되었을 때, 해당 웨이퍼가 어떤 조건에서 공정되었는지 역추적하는 것은 원인 분석의 핵심입니다. 플랫폼의 공정 이력(Equipment History) 기능은 이를 위해 모든 PJOB(Process Job)의 시작과 종료, 상태 정보를 기록합니다.

## 4.2 레시피 구조 및 이력 조회
플랫폼에 등록된 레시피 목록을 통해 현재 운용 중인 공정 조건을 확인할 수 있습니다.
- **주요 레시피 예시**: `RUN\0_P_IH_HD02`, `RUN\0_P_IH_HD02-1` 등
- **추적 항목**: 각 레시피 수행 시의 시작 시간, 종료 시간, 그리고 최종 종료 상태(Success/Fail)를 저장하여 공정의 안정성을 평가합니다.

## 4.3 데이터 기반의 원인 분석 프로세스
특정 시점에 이상이 발생했다면, 다음과 같은 단계로 데이터를 분석합니다.
1. **레시피 특정**: 이상이 발생한 시점의 PJOB이 어떤 레시피(`RUN\0_P_IH_HD02` 등)로 수행되었는지 확인합니다.
2. **이력 비교**: 동일한 레시피로 수행된 이전 작업들과 현재 작업의 파라미터 추이를 비교합니다.
3. **상관 분석**: 앞선 2장에서 분석한 `MFC8_NH3`의 이상 시점이 특정 레시피의 특정 단계(`N-FL1`)에서 반복적으로 나타나는지 확인하여 레시피 자체의 문제인지 설비의 문제인지 판별합니다.

## 4.4 이력 데이터를 통한 공정 최적화
축적된 이력 데이터는 단순히 문제 해결뿐만 아니라 공정 최적화에도 활용됩니다. 성공적으로 완료된 PJOB들의 파라미터 분포를 분석하여 'Golden Recipe'의 범위를 설정하고, 이를 기준으로 공정 윈도우(Process Window)를 최적화함으로써 수율 향상을 도모할 수 있습니다.
