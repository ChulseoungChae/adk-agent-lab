# 5. 제 5장: 통계 리포트 분석: 데이터 기반의 월간 공정 인사이트 추출

# 제 5장: 통계 리포트 분석: 데이터 기반의 월간 공정 인사이트 추출

반도체 공정의 안정성을 유지하기 위해서는 개별적인 이상 현상을 파악하는 것만큼이나, 장기간에 걸친 데이터의 트렌드를 분석하는 것이 중요합니다. 본 장에서는 AI 어시스턴트 플랫폼이 제공하는 **통계 리포트(Monthly/Daily Report)** 기능을 통해 공정의 건강 상태를 거시적인 관점에서 진단하는 방법을 다룹니다.

## 1. 월간 리포트의 역할과 구성
플랫폼은 주기적으로 이상감지 로그를 집계하여 리포트를 생성합니다. 예를 들어, `report_monthly_2026-01` 리포트는 한 달 동안 발생한 모든 이상 사례를 통계적으로 요약하여 제공합니다.

### 리포트의 핵심 지표
- **총 이상 발생 건수 (Total Logs):** 해당 기간 내 발생한 모든 이상 신호의 합계입니다.
- **파라미터별 발생 빈도 (Parameter Stats):** 어떤 센서나 유량계(MFC)에서 문제가 자주 발생했는지 보여줍니다.
- **이상 유형별 분포 (Violation Type Stats):** 상한 초과(Upper Limit), 하한 미달(Lower Limit), 예측 오차 발생 등 문제의 성격을 분류합니다.

## 2. 실제 데이터 사례 분석
`report_monthly_2026-01` 리포트의 데이터를 분석해보면 다음과 같은 구체적인 인사이트를 얻을 수 있습니다.

### 주요 파라미터 이상 통계
| 파라미터 명 | 이상 발생 횟수 | 주요 위반 유형 |
| :--- | :---: | :--- |
| **MFC1_N2-1** | 4회 | 상한 초과, 하한 미달 |
| **MFC3_N2-3** | 4회 | 상한 초과, 하한 미달 |
| **MFC8_NH3** | 4회 | 상한 초과 |
| **MFC2_N2-2** | 3회 | 상한 초과, 하한 미달 |
| **MFC4_N2-4** | 3회 | 상한 초과 |

### 위반 유형 분석 (Violation Type)
리포트 데이터에 따르면, 전체 18건의 로그 중 **유형 1(상한 초과)**이 7건으로 가장 높은 비중을 차지했습니다. 이는 특정 가스 유량의 제어가 불안정해지는 경향이 있음을 시사합니다. 반면, **유형 4(하한 미달)**는 3건으로 나타나, 유량이 급격히 떨어지는 현상에 대해서는 상대적으로 관리가 잘 되고 있음을 알 수 있습니다.

## 3. 엔지니어를 위한 활용 가이드
통계 리포트는 단순한 숫자의 나열이 아니라, **설비 유지보수(PM)의 우선순위**를 결정하는 근거가 됩니다.

1. **반복적 위반 파라미터 식별:** `MFC8_NH3`와 같이 특정 파라미터에서 상한 초과가 지속적으로 발생한다면, 해당 라인의 MFC 밸브나 제어 로직을 즉시 점검해야 합니다.
2. **트렌드 추적:** 월간 리포트의 발생 건수 추이를 비교하여, 공정 산포가 점진적으로 커지고 있는지 아니면 특정 시점에 집중되었는지를 판단하여 예방 정비 주기를 조절할 수 있습니다.
3. **의사결정 자동화:** 누적된 통계 데이터를 바탕으로, 이상 빈도가 임계치를 넘을 경우 자동으로 설비 가동 중단(Interlock)을 검토하는 지능형 운영 체계를 구축할 수 있습니다.

결론적으로, 플랫폼의 리포트 기능은 파편화된 로그 데이터를 의미 있는 **공정 지식(Domain Knowledge)**으로 변환하여, 엔지니어가 데이터에 기반한 정확한 판단을 내릴 수 있도록 돕습니다.
