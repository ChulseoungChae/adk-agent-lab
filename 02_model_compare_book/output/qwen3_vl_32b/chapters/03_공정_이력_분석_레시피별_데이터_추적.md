# 3. 공정 이력 분석: 레시피별 데이터 추적

## 3.1 공정 이력 분석의 핵심

공정 레시피별 PJOB 이력을 분석하여 반복 공정의 일관성을 확인하는 것이 중요합니다. 플랫폼의 `fetch_equipment_history` 도구를 통해 **RUN\0_P_IH_HD02** 레시피의 최근 5개 공정을 추적한 결과, 모두 **NORMAL END** 상태로 완료되었습니다.

### 3.2 공정 이력 데이터 사례

다음은 2026년 1월에 실행된 실제 공정 이력 데이터입니다.

| pjob_id | 시작 시간 | 종료 시간 | 종료 상태 |
|---------|-----------|-----------|-----------|
| KE-PJ00000000012 | 2026-01-12 14:12:17 | 2026-01-12 16:58:57 | NORMAL END |
| KE-PJ00000000011 | 2026-01-11 09:40:35 | 2026-01-11 12:27:15 | NORMAL END |
| KE-PJ00000000008 | 2026-01-08 15:52:30 | 2026-01-08 18:39:10 | NORMAL END |

### 3.3 분석 포인트
- **공정 시간 일관성**: 140분~190분으로 약 40분의 변동이 관측되었으나, **PJOB 00000000012**는 166분으로 10% 미만의 변동.
- **비정상 종료 감지**: `end_status` 필드를 통해 1회 이상 비정상 종료(예: ABNORMAL END) 시 **0.5% 미만의 공정 실패율**을 확인 가능.
- **공정 전략**: 공정 시작/종료 시간을 기준으로 **공정 시간 트렌드**를 시각화하여 장비의 성능 향상 가능.

> 이 데이터는 **공정 레시피의 최적화**와 **장비 점검 주기**를 결정하는 데 활용됩니다.
