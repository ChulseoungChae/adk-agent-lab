# 5장 플랫폼 통합 활용: 데이터 생성기부터 리포트 생성까지

## 5.1 플랫폼 전체 아키텍처 overview

반도체 AI 어시스턴트 플랫폼은 데이터 수집부터 리포트 생성까지 통합된 워크플로우를 제공합니다. 각 구성 요소가 어떻게 연동되는지 살펴보겠습니다.

### 플랫폼 구성 요소
```
[데이터 생성기] → [실시간 이상감지] → [예측 모델] → [리포트 생성] → [엔지니어 확인]
     (Generator)        (Rule 기반)       (PatchTST)      (Daily/Monthly)    (Dashboard)
```

## 5.2 데이터 생성기 상태 관리

데이터 생성기(Genertor)는 공정 데이터 수집의 첫 단계입니다. 현재 생성기 상태는 다음과 같습니다.

### 현재 생성기 상태
| 항목 | 값 |
|------|-----|
| 상태(Status) | RUN |
| 마지막_tick | 2026-06-17T16:24:11 |

생성기가 **RUN** 상태이면 실시간 데이터 수집이 정상적으로 이루어지고 있음을 의미합니다. STOP 상태일 경우 이상감지 및 리포트 생성이 중단되므로, 정기적인 상태 확인이 필수입니다.

## 5.3 이상감지 시스템 연동

실시간 이상감지(1장)와 예측 이상감지(2장)는 생성기에서 수집된 데이터를 기반으로 다음과 같이 연동됩니다.

### 실시간 Rule 기반 이상감지
- 상한선/하한선 초과 시 즉시 경고
- MFC8_NH3, MFC3_N2-3 등 주요 파라미터 모니터링
- is_interrupted 값으로 공정 중단 심각도 판단

### 예측 모델 기반 이상감지
- PatchTST 예측값과 실제값 비교
- 편차가 30% 이상일 때 이상 판정 (예상)
- 패턴 기반 이상 조기 탐지

## 5.4 리포트 생성 워크플로우

### 리포트 생성 방법
```
플랫폼 API: create_platform_report(report_type='daily', days_ago=1)
```

- **일간 리포트**: days_ago=1 (어제), days_ago=2 (그저께) 등 과거 날짜 지정
- **월간 리포트**: report_type='monthly'로 생성, 자동 월간 기간 적용

### 리포트 메타데이터 확인
```
플랫폼 API: list_platform_reports(limit=10)
```
생성된 리포트 목록을 확인하고, 필요시 get_report_content(report_id)로 상세 내용 조회 가능합니다.

## 5.5 엔지니어의 일상적 운영 체크리스트

### 매일 확인
1. 생성기 상태 (RUN 여부)
2. 전일 일간 리포트 이상감지 건수
3. 주요 이상 파라미터 TOP 3

### 매주 확인
1. 해당 주 이상 발생 추세 (상승/감소)
2. 동일 레시피(Normal End 여부) 반복 확인
3. 예측 모델 이상 vs Rule 기반 이상 비교

### 매월 확인
1. 월간 리포트 전체 이상률 분석
2. 레시피별/파라미터별 이상률 비교
3. 다음 달 공정 개선 계획 수립

## 5.6 플랫폼 활용의 장단점

### 장점
- **실시간 모니터링**: Rule 기반 이상감지로 즉시 대응 가능
- **정밀도 향상**: PatchTST 예측 모델로 조기 탐지
- **자동화**: 리포트 자동 생성으로 엔지니어 업무 부담 감소
- **데이터 기반 의사결정**: 객관적 통계 기반 공정 개선

### 단점 및 극복 방안
- **과도한 경보**: Rule 기반 이상감지의 False Positive → 예측 모델과 함께 활용
- **학습 데이터 의존**: PatchTST 정확도 → 양질의 공정 데이터 확보 필요
- **시스템 가용성**: 생성기 STOP 시 모든 기능 중단 → 정기적 상태 확인 필수

## 5.7 결론

반도체 AI 어시스턴트 플랫폼은 실시간 이상감지부터 리포트 생성까지 통합된 솔루션을 제공합니다. 엔지니어는 생성기 상태 확인, 일간/월간 리포트 검토, 이상 사례 교차 분석을 통해 공정 안정성을 지속적으로 개선할 수 있습니다. 1~4장에서 실제 데이터를 통해 본 사례들이 실제 운영에 바로 활용되기를 바랍니다.
