# 제4장: 공정 이력 데이터 분석을 통한 설비 상태 파악 및 추세 관리

반도체 제조 공정, 특히 ALD(Atomic Layer Deposition)와 PVD(Physical Vapor Deposition)와 같은 정밀 증착 공정에서 설비의 상태를 파악하는 것은 수율(Yield) 확보의 핵심입니다. 단순한 실시간 모니터링을 넘어, 과거의 공정 이력(Equipment History)을 분석하는 것은 설비의 노후화, 부품 교체 주기, 그리고 반복적인 성능 저하 패턴을 찾아내는 데 결정적인 역할을 합니다.

## 1. PJOB 이력 분석의 중요성

본 플랫폼은 각 공정 레시피별로 실행된 `PJOB ID`와 실행 시간, 종료 상태를 체계적으로 기록합니다. 예를 들어, `RUN\\0_P_IH_HD02` 레시피의 실행 이력을 살펴보면 다음과 같은 데이터를 확인할 수 있습니다.

| PJOB ID | 레시피 명 | 시작 시간 | 종료 시간 | 종료 상태 |
| :--- | :--- | :--- | :--- | :--- |
| KE-PJ00000000012 | RUN\\0_P_IH_HD02 | 2026-01-12 14:12:17 | 2026-01-12 16:58:57 | NORMAL END |
| KE-PJ00000000011 | RUN\\0_P_IH_HD02 | 2026-01-11 09:40:35 | 2026-01-11 12:27:15 | NORMAL END |
| KE-PJ00000000008 | RUN\\0_P_IH_HD02 | 2026-01-08 15:52:30 | 2026-01-08 18:39:10 | NORMAL END |

이러한 데이터는 단순한 로그 그 이상의 가치를 지닙니다. 엔지니어는 `process_start_time`과 `process_end_time`의 차이를 분석하여 공정 시간(Process Time)의 변동성을 측정할 수 있습니다. 만약 특정 시점 이후부터 공정 시간이 일정하게 길어지는 패턴이 발견된다면, 이는 챔버 내의 파티클 증가나 펌핑 시스템의 성능 저하를 암시하는 강력한 신호가 될 수 있습니다.

## 2. 레시피별 성능 추세 관리

공정 이력 데이터는 특정 레시피의 안정성을 평가하는 데 사용됩니다. 플랫폼은 `search_process_recipes`를 통해 등록된 모든 레시피를 식별하고, 각 레시피에 해당하는 이력을 정렬하여 보여줍니다.

엔지니어는 다음과 같은 분석 시나리오를 수행할 수 있습니다:

1.  **Cycle Time 변동성 분석**: 동일한 레시피가 실행될 때 소요되는 시간의 표준 편차를 계산하여 공정의 정밀도를 평가합니다.
2.  **종료 상태(End Status) 모니터링**: `NORMAL END`가 아닌 `ERROR`나 `ABORT` 상태가 빈번하게 발생하는 레시피나 특정 설비를 식별하여 선제적인 유지보수(PM) 계획을 수립합니다.
3.  **설비 간 비교 분석**: 동일한 레시피를 수행하는 서로 다른 챔버(Chamber) 간의 이력 데이터를 비교하여, 특정 챔버에서만 발생하는 성능 편차를 찾아냅니다.

## 3. 데이터 기반의 예방 보전(Preventive Maintenance)

과거 데이터를 통한 추세 관리는 '사후 대응'에서 '사전 예방'으로의 전환을 의미합니다. 예를 들어, `PJOB`의 실행 시간이 점진적으로 증가하는 추세가 관찰된다면, 이는 부품의 교체 시기가 도래했음을 알려주는 지표입니다. 

AI 플랫폼은 이러한 이력 데이터를 시계열로 분석하여, 설비가 임계치에 도달하기 전에 엔지니어에게 알람을 제공합니다. 이를 통해 갑작스러운 설비 중단(Down-time)으로 인한 생산 손실을 최소 가로막고, 계획된 일정에 따라 부품을 교체함으로써 공정의 연속성을 보장할 수 있습니다.

결론적으로, 공정 이력 데이터의 체계적인 관리와 분석은 설비의 생애 주기(Life Cycle)를 관리하고, 반도체 제조 공정의 안정성을 극대화하는 데 필수적인 요소입니다.
