# 5. 제5장: 데이터 기반 리포팅 - 월간/일간 통계 분석을 통한 인사이트 도출

# 제5장: 데이터 기반 리포팅 - 월간/일간 통계 분석을 통한 인사이트 도출

반도체 공정 엔지니어에게 가장 중요한 과제 중 하나는 방대한 양의 공정 데이터 속에서 유의미한 패턴을 찾아내는 것입니다. 단순히 개별적인 이상 징후를 포착하는 것을 넘어, 특정 기간 동안 발생한 이상 현상의 빈도와 유형을 분석함으로써 설비의 노후화나 공정 레시피의 불안정성을 파악할 수 있어야 합니다.

본 장에서는 AI 어시스턴트 플랫폼이 제공하는 **자동화된 통계 리포트 기능**을 통해 어떻게 데이터 기반의 의사결정을 내릴 수 있는지 살펴보겠습니다.

## 1. 자동 생성 리포트의 구조와 활용

플랫폼은 `daily` 및 `monthly` 단위로 이상감지 통계를 집계하여 리포트를 생성합니다. 이 리포트는 단순히 숫자의 나열이 아니라, 공정 엔지니어가 집중해야 할 타겟을 명확히 제시합니다.

### 리포트 구성 요소
- **전체 이상 로그 수 (Total Logs):** 특정 기간 내 발생한 전체 이상 징후의 규모를 파악합니다.
- **파라미터별 발생 빈도 (Parameter Stats):** 어떤 MFC(Mass Flow Controller)나 압력 센서에서 문제가 빈번한지 식별합니다.
- **위반 유형별 통계 (Violation Type Stats):** 단순 임계치 초과인지, 예측 모델과의 괴리인지 등을 분류합니다.
- **상세 위반 분석 (Parameter Violation Stats):** 각 파라미터별로 어떤 유형의 위반(Upper/Lower Limit Violation 등)이 발생했는지 세부적으로 보여줍니다.

## 2. 케이스 스터디: 2026년 1월 월간 리포트 분석

실제 플랫폼에 저장된 `report_monthly_2026-01` 데이터를 바탕으로 분석 사례를 살펴보겠습니다.

### [데이터 분석 결과]
| 분석 항목 | 상세 내용 |
| :--- | :--- |
| **분석 기간** | 2026-01-01 ~ 2026-02-01 |
| **총 이상 발생 건수** | 18건 |
| **주요 집중 관리 파라미터** | `MFC1_N2-1`, `MFC3_N2-3`, `MFC8_NH3` (각 4건) |
| **주요 위반 유형** | Type 1 (7건), Type 2 (5건), Type 3 (3건), Type 4 (3건) |

### 상세 분석 및 엔지니어의 액션 아이템

#### (1) MFC 유량 제어 불안정성 포착
리포트에 따르면 `MFC1_N2-1`, `MFC3_N2-3`, `MFC8_NH3` 세 가지 파라미터에서 각각 4건의 이상이 발생했습니다. 특히 `MFC1_N2-1`의 경우, 상한 임계치 초과(Type 1: 2건), 하한 임계치 미달(Type 4: 1건), 그리고 예측 모델과의 불일치(Type 2: 1건)가 동시에 관찰되었습니다.

이는 단순히 특정 시점의 일회성 오류가 아니라, **해당 MFC의 응답 속도가 느려지거나 밸브의 응답성이 저하되고 있음**을 시사합니다. 엔지니어는 해당 MFC의 교체 주기 검토나 캘리브레이션(Calibration)을 우선순위로 진행해야 합니다.

#### (2) 위반 유형(Violation Type)에 따른 진단
- **Type 1 (Upper Limit Violation):** 유량이 설정값보다 높게 튀는 현상. 가스 공급 라인의 압력 서지(Surge) 가능성 검토 필요.
- **Type 2 (Lower Limit Violation):** 유량이 설정값보다 낮게 떨어지는 현상. 가스 공급 라인의 막힘(Clogging)이나 밸브 폐쇄 문제 의심.
- **Type 4 (Prediction Mismatch):** 예측값과 실제값의 큰 차이. 이는 설비의 물리적 상태 변화가 급격히 일어나고 있음을 의미하며, 센서의 드리프트(Drift) 현상을 의심할 수 있습니다.

## 3. 리포트를 통한 선제적 유지보수 전략

리포트 분석의 궁극적인 목적은 **'사후 조치'에서 '사전 예방'으로의 전환**입니다. 

1. **트렌드 추적:** 월간 리포트의 `total_logs` 수치가 매달 증가하고 있다면, 이는 설비 전반의 노후화가 진행 중임을 의미합니다.
2. **반복적 문제 식별:** 특정 파라미터가 매달 리포트 상위권에 등장한다면, 이는 단순한 노이뮬레이션 오류가 아닌 설비 구조적 결함(Structural Issue)으로 판단하고 근본 원인 분석(RCA)에 착수해야 합니다.

결론적으로, AI 플랫폼의 리포팅 기능은 엔지니어에게 수만 개의 로우 데이터(Raw Data)를 일일이 검토하는 수고를 덜어주는 동시에, 통계적 근거를 바탕으로 한 강력한 설비 관리 도구를 제공합니다.
